# KDBScope Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:executing-plans (inline execution — user
> requested autonomous single-session build; author == executor, so per-step code listings are
> elided in favor of exact interfaces and verification gates). Steps use checkbox syntax.

**Goal:** Build KDBScope — a Dockerized microservices tool that indexes kdb/ logs, Claude Code
session JSONL, git history, and docs across all projects, searchable via web UI, CLI, REST, MCP.

**Architecture:** 7-service compose stack (indexer, api, mcp, ui, qdrant, redis, postgres).
Node 22 TS monorepo (npm workspaces): `packages/core` (all domain logic), thin service wrappers,
React UI. Hybrid dense+sparse RRF search in Qdrant; Postgres catalog; BullMQ jobs.

**Tech Stack (pinned, resolved 2026-07-09 via `npm view <pkg> version`):**
typescript 7.0.2 (lint via tsc) — NOTE: use 5.9.x if 7.x breaks tooling; hono 4.12.28;
@hono/node-server 2.0.8; bullmq 5.79.3; ioredis 5.11.1; pg 8.22.0; @qdrant/js-client-rest 1.18.0;
@modelcontextprotocol/sdk 1.29.0; commander 15.0.0; react/react-dom 19.2.7; vite 8.1.3;
@vitejs/plugin-react 6.0.3; tailwindcss/@tailwindcss/vite 4.3.2; vitest 4.1.10; zod 4.4.3;
node-cron 4.6.0.

## Global Constraints

- Node >= 22; PostgreSQL >= 18 (org baseline §10); all images pinned exact version.
- All KDB-of-this-project writes via blessed helper `bin/kdb_append` (§2.6).
- Tests under `test/` at repo root (§3.6). No hard-coded config values (§3.1).
- Read-only bind mounts for `~/.claude` and `__CODING NEW`; tool never writes to indexed projects.
- Commits: casual human tone, categorized, no AI/co-author mentions (§11).
- Ports: api 8710, mcp 8711, ui 8712, qdrant 6363/6364, redis 6390, postgres 5460 (127.0.0.1).
